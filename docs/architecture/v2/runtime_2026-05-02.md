# Flark v2 Runtime

Status date: 2026-05-02

## Scope

`FlarkEditorRuntime` is the first headless composition layer. It keeps the
core model practical without moving history, command registry, or extension
state into `FlarkEditorState`.

## Contract

- Runtime state is immutable.
- `FlarkEditorState` stays focused on document plus selection.
- `FlarkHistoryStack` stays a companion object.
- `FlarkCommandRegistry` is built from registered extensions.
- Command dispatch returns a `FlarkEditorRuntimeResult`.
- Handled commands with transactions apply the transaction and record history.
- Rejected or unhandled commands leave the runtime unchanged.
- `undo()` and `redo()` are runtime operations, not document-state operations.

## Why This Matters

The runtime is the future home for composition that v1 currently spreads across
`TextEditingController`, command objects, and widget adapters. This keeps the
new architecture honest: Flutter can adapt to a runtime, but the runtime has no
Flutter dependency.

## Next Runtime Work

- Add runtime-owned extension state after extension APIs need state.
- Add command/history result helpers once command surfaces expand.
- Keep parse/projection invalidation metadata flowing through new command and

- Add performance counters outside correctness state.
